Security & Trust

How Elite Web Services protects your AWS environment during assessments and advisory engagements.

Security Practices

Security is embedded in every engagement — not bolted on at the end. We follow AWS Well-Architected security pillar guidelines, enforce least-privilege access, and maintain audit trails for all cross-account operations.

AWS Integration Approach

Our portal uses AWS STS AssumeRole with a cross-account IAM role. We request read-only permissions only — no write access, no access keys stored, and no persistent credentials on our servers. Every customer connection uses a unique External ID to prevent the confused deputy problem.

Data Handling Overview

What We Access

Cost Explorer billing data, resource inventory metadata, Security Hub findings, and Well-Architected review data — configuration only, never your data payloads.

What We Never Access

Database contents, S3 objects, Secrets Manager values, IAM passwords, or the ability to create, modify, or delete resources in your account.

AI Insights

Claude AI generates cost and architecture insights from aggregated, non-sensitive metadata — never raw credentials or customer data payloads.

Retention

Assessment data is retained only for the duration of the engagement. Customers can revoke cross-account access at any time by removing the IAM trust policy.
